## Installation

1.  **Download** the package for your operating system.
2.  **Run the installer**:
    *   **Windows**: Execute the `.exe` and follow the prompts.
    *   **macOS**: Open the `.dmg` and move Lokinode to Applications.
    *   **Linux**: Install via `sudo dpkg -i lokinode_*.deb`.
3.  **Setup**: Follow the on-screen instructions to create a new 24-word seed phrase or import an existing one.

## Capabilities

### Node Monitoring
Monitor synchronization status, block height, and mempool activity. The dashboard provides real-time updates on peer connections and network health.

### Wallet Operations
Manage on-chain FLC balances. The interface supports sending and receiving payments with built-in fee estimation and comprehensive transaction history.

### Channel Management
Interact with the Lightning Network. Users can open and close channels, monitor liquidity, and manage invoices within the GUI.

## Data Locations

Lokinode stores node data, logs, and channel databases in the default FLND directory:

| OS | Default Path |
| :--- | :--- |
| **Linux** | `~/.flnd/` |
| **macOS** | `~/Library/Application Support/Flnd/` |
| **Windows** | `%APPDATA%\Flnd\` |

## CRITICAL: Lightning Backups

A 24-word seed phrase **does not recover Lightning channel funds**. It only recovers your on-chain balance.

1.  **Static Channel Backups (SCB)**: Your node generates a `channel.backup` file (located in `.flnd/data/chain/flokicoin/mainnet/`). This file is required to recover funds from open channels.
2.  **Requirement**: To fully restore a node, you must have **both** the 24-word seed and the most recent `channel.backup` file.

:::danger[DATA LOSS RISK]
Automate the backup of your `channel.backup` file. Without it, funds in Lightning channels cannot be recovered if local data is lost.
:::
