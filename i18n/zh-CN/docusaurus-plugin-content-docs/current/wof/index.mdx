## Web2、Web3……以及下一代网络

- **Web1** 让我们能够发布静态页面，但表达方式十分有限。
- **Web2** 叠加了丰富的界面，然而每次登录、每条评论、每个数据集都汇入了渴望获取我们数据的封闭平台。
- **比特币** 展示了另一条路——价值可以在无中间商的情况下流通。
- **Web3** 让钱包和代币走向主流，但大多只是围绕同样的中间商发行资产；日常网络互动从未真正改变。

WoF 是重置按钮。我们聚焦于日常互联网服务——地图、通讯、媒体、商务——并以透明的经济机制、数据所有权、主权身份、凝聚力协作和可量化的参与度重新构想它们。目标不是装饰旧网络，而是发明一个更人性化的网络。

## 为什么 Nostr + PoW（工作量证明）是 WoF 的支柱

大多数人听到"[Nostr](https://jump.ohstr.com/)"时会联想到由 Jack Dorsey 推广的去中心化社交应用。实际上，Nostr 是一个轻量级通信层——可将其理解为下一代网络的 HTTP：简单、开放且可移植。PoW（工作量证明）是保持下一代网络诚实的经济层。您要么花费此前赚取的工作量支撑的代币来访问服务，要么贡献新的工作量以换取其他人可以依赖的价值。

WoF 依托的核心原则：

- **工作量证明**：服务以能源支撑的代币定价访问权——用户可以花费此前挖矿所得，或承担新的工作来获得准入。
- **密钥即身份**：公/私钥对取代了平台绑定的用户名，使声誉具备可移植性。
- **签名笔记**：每次互动都是不可篡改、带有时间戳的事件，任何客户端均可读取、验证或扩展。
- **中继选择**：参与者可以选择或自行运行中继来路由事件，因此没有任何单一中介控制通信。

WoF 基于 Nostr 构建，使每项服务从第一天起就使用同一种语言；基于工作量证明构建，使每个经济承诺都可追溯到真实努力。通信、协调和参与通过 Nostr 传输，而定价、访问和治理则依托可验证的工作。


## 什么是 FEP？

**FEP** 代表 **Fun Enhancement Possibility（趣味增强可能性）**。
FEP 是 WoF 的提案蓝图。它阐明您希望改善的体验问题、您邀请人们踏上的旅程，以及将支撑其运转的经济机制。

它不是功能需求。
它不是拉盘方案。
它是一项设计挑战，邀请任何人重新定义网络规则。

一份清晰的 FEP 通常涵盖：

- 当前现状以及您希望重新设计的内容。
- 当前涉及哪些参与者，以及他们的体验将如何改善。
- 经济设计：价值如何流通，什么工作支撑它，以及参与者如何保持一致。
- 服务所依赖的 Nostr 基元或新的 NIP 草案。

它不是路线图——它是发射台。提交您的 FEP，让我们共同出发。

## 为什么要在现有基础上构建，而非对抗它？

我们并非要摧毁现有的网络。人们每天依赖的习惯、网络和工具，正是通往下一阶段的发射台。WoF 实验扩展现有的可行之物，而不是强迫所有人一夜之间迁移。

我们选择在现有基础上构建，因为：

- **熟悉的入口**让人们无需放弃现有工作流程即可尝试新体验。
- **渐进式升级**让团队能够比从头重写更快地交付、衡量和迭代。
- **互操作性**保持了与旧有服务之间的桥梁畅通。

这就是为什么 WoF 项目与当今的服务对接，通过 Nostr 添加透明通信，并融入以工作量为支撑的激励机制。演进胜于孤立。
