Lokinode est une application de bureau multiplateforme qui fournit une interface graphique pour l'exécution et la gestion d'un nœud complet Flokicoin (FLND). Elle vous permet d'exploiter un portefeuille on-chain et de participer au réseau Flokicoin Lightning (FLN).

## Caractéristiques

- **Service en arrière-plan** : Lokinode fonctionne comme un service persistant. Vous pouvez fermer l'interface utilisateur pendant que le nœud continue de s'exécuter dans la barre d'état système.
- **Portefeuille on-chain** : Initialisez et déverrouillez votre portefeuille, envoyez/recevez des FLC et consultez l'historique des transactions.
- **Gestion des nœuds** : Surveillez la hauteur des blocs, l'état de synchronisation et le tip du mempool. Visualisez les journaux de l'application directement dans l'interface.
- **Profils multi-nœuds** : Exécutez et gérez plusieurs nœuds sur la même machine. Lokinode suit automatiquement vos répertoires de données et vos configurations, vous permettant de basculer rapidement entre différents profils de nœuds.

## Installation et configuration

:::tip[Vous venez de tWallet ?]
Si vous utilisez déjà [**tWallet**](/wallets/twallet), **aucune migration n'est nécessaire**. Votre phrase de récupération de 24 mots est entièrement compatible. Mieux encore, comme les deux applications partagent les mêmes répertoires de données par défaut, Lokinode détectera automatiquement votre nœud existant. Vous pouvez simplement lancer l'application et activer votre profil de nœud par défaut avec **aucune configuration requise**.
:::

1. **Téléchargement** : Téléchargez l'exécutable correspondant à votre système d'exploitation.
2. **Configuration du nœud** : Lancez Lokinode. L'assistant de configuration vous demandera d'abord de définir un alias public et de sélectionner un répertoire de données local.
3. **Configuration du portefeuille** : Choisissez soit de créer un nouveau portefeuille (qui fournit une nouvelle phrase de récupération de 24 mots), soit de restaurer un portefeuille existant via mnémonique.
4. **Sécurité** : Définissez un mot de passe de cryptage fort pour protéger vos données locales.
5. **Synchronisation** : Une fois le nœud démarré, il commencera automatiquement à se synchroniser avec la blockchain Flokicoin.

:::info[Pour les opérateurs de nœuds et les développeurs]
Vous cherchez à déployer un nœud headless sur un VPS, à exécuter des opérations automatisées (Ops) ou à l'intégrer avec Docker ? Vous préférerez peut-être exécuter le démon [**`flnd`**](/wallets/flnd) directement au lieu de l'interface graphique de Lokinode. `flnd` fournit le backend central et peut être géré entièrement via la ligne de commande.
:::

## Gestion avec Lokihub

Alors que Lokinode s'occupe de l'exécution de votre nœud Lightning en arrière-plan et fournit une interface pour les opérations de base du portefeuille on-chain, vous devez utiliser **Lokihub** pour les fonctionnalités avancées du réseau Lightning.

Lorsque vous ouvrez Lokihub sur la même machine, il tentera automatiquement de se connecter à votre répertoire Lokinode actif, débloquant les capacités suivantes :

- **Gestion de la liquidité** : Ouvrez, fermez et surveillez les canaux Lightning pour gérer votre capacité entrante et sortante.
- **Sous-portefeuilles** : Créez des portefeuilles dédiés pour organiser vos fonds pour différentes applications ou budgets personnels.
- **Intégrations d'applications (NWC)** : Connectez de manière transparente des applications externes à votre nœud à l'aide de Nostr Wallet Connect, avec des contrôles de permissions granulaires.

[Voir le guide de configuration de Lokihub →](/lokihub/setup)

## Emplacements des données

Par défaut, Lokinode stocke les données de la blockchain et du portefeuille dans les répertoires suivants :

| OS | Chemin par défaut |
| :--- | :--- |
| **Linux** | `~/.flnd/` |
| **macOS** | `~/Library/Application Support/Flnd/` |
| **Windows** | `%APPDATA%\Flnd\` |

Au-delà de ces chemins par défaut, vous pouvez créer un nouveau profil dans un emplacement personnalisé ou déplacer un profil existant en copiant son répertoire de données vers un nouveau chemin et en le rouvrant via la gestion des profils de l'application.

## CRITIQUE : Sauvegardes Lightning

Votre phrase de récupération de 24 mots **ne récupère que les fonds on-chain**.

Pour récupérer les fonds verrouillés dans les canaux Lightning ouverts, vous avez besoin d'une **sauvegarde statique des canaux (SCB)**.

:::danger[RISQUE DE PERTE DE DONNÉES]
Assurez-vous toujours d'avoir une sauvegarde séparée et vérifiée de votre fichier `channel.backup` aux côtés de votre phrase de récupération. Sans cela, les fonds des canaux Lightning ne peuvent pas être récupérés en cas de perte de données locales.
:::
