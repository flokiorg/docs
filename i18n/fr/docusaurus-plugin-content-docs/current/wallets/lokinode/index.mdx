## Installation

1.  **Téléchargez** le paquet pour votre système d'exploitation en utilisant les boutons ci-dessus.
2.  **Exécutez l'installateur** :
    *   **Windows** : Lancez le `.exe` et suivez l'assistant de configuration.
    *   **macOS** : Ouvrez le `.dmg` et faites glisser Lokinode vers votre dossier Applications.
    *   **Linux** : Installez le paquet `.deb` via votre gestionnaire de paquets.
3.  **Configuration** : Lancez l'application pour initialiser votre nœud. Vous pouvez créer une nouvelle phrase de récupération de 24 mots ou importer un portefeuille existant.

## Capacités

### Opérations du Lightning Network (FLN)
Lokinode simplifie la gestion avancée du Lightning Network :
*   **Contrôle visuel des canaux** : Ouvrez, fermez et surveillez l'état des canaux sans commandes de terminal.
*   **Gestion de la liquidité** : Suivez la capacité entrante et sortante avec un retour visuel en temps réel.
*   **Règlement** : Générez et payez des factures Lightning avec une confirmation instantanée sur le réseau.

### Expérience persistante
Conçu pour la fiabilité du nœud, Lokinode sépare le processus du nœud de l'interface graphique :
*   **Intégration de la barre d'état système** : Contrôlez l'état du nœud et visualisez les mises à jour rapides de l'état depuis la barre d'état du système d'exploitation.
*   **Exécution non intrusive** : La fermeture de la fenêtre de l'application masque l'interface mais maintient le nœud actif en arrière-plan.
*   **Auto-stabilité** : Logique intégrée pour gérer les interruptions de réseau et maintenir la connectivité avec les pairs.

### Configuration flexible
Lokinode est conçu pour les utilisateurs avancés qui exigent un contrôle sur leur environnement :
*   **Chemins personnalisés** : Spécifiez n'importe quel répertoire local ou externe pour le stockage des données de la blockchain et des canaux.
*   **Métriques en temps réel** : Surveillez la hauteur des blocs, la progression de la synchronisation et le nombre de pairs directement depuis le tableau de bord.
*   **Suite de sécurité** : Gérez le stockage de portefeuilles chiffrés et les informations de récupération dans un environnement local sécurisé.

## Gestion avec Lokihub

Alors que **Lokinode** fournit une interface intégrée pour les opérations de base, les opérateurs avancés peuvent utiliser **Lokihub** pour des analyses de routage détaillées et une gestion de nœuds à distance.

### Connexion rapide
1.  **Nœud actif** : Assurez-vous que Lokinode est en cours d'exécution et synchronisé.
2.  **Se connecter** : Ouvrez Lokihub ; il tentera automatiquement de détecter les informations d'identification de votre répertoire Lokinode actif.
3.  **Synchroniser** : Accédez à des graphiques de liquidité avancés et à l'historique de routage.

## Emplacements des données

Lokinode utilise des chemins standard par défaut, mais permet une personnalisation complète lors de la configuration initiale ou via les paramètres.

| OS | Chemin par défaut |
| :--- | :--- |
| **Linux** | `~/.flnd/` |
| **macOS** | `~/Library/Application Support/Flnd/` |
| **Windows** | `%APPDATA%\Flnd\` |

## CRITIQUE : Sauvegardes Lightning

Votre phrase de récupération de 24 mots **ne récupère que les fonds on-chain**. 

1.  **Sauvegardes statiques de canaux (SCB)** : Un fichier `channel.backup` est nécessaire pour récupérer les fonds des canaux Lightning ouvers.
2.  **Exigence de sécurité** : Vous devez maintenir une sauvegarde séparée du fichier `channel.backup` aux côtés de votre phrase de récupération.

:::danger[RISQUE DE PERTE DE DONNÉES]
Assurez-vous toujours d'avoir une sauvegarde vérifiée de votre fichier `channel.backup`. Sans cela, les fonds des canaux Lightning ne peuvent pas être récupérés si les données locales sont perdues.
:::
