Lokinode est une application de bureau multiplateforme qui fournit une interface graphique pour exécuter et gérer un nœud complet Flokicoin (FLND). Il vous permet d'opérer un portefeuille on-chain et de participer au Flokicoin Lightning Network (FLN).

## Fonctionnalités

- **Service en arrière-plan** : Lokinode s'exécute comme un service persistant. Vous pouvez fermer l'interface utilisateur pendant que le nœud continue de fonctionner dans la barre d'état système.
- **Portefeuille On-chain** : Initialisez et déverrouillez votre portefeuille, envoyez/recevez des FLC et consultez l'historique des transactions.
- **Gestion de nœud** : Surveillez la hauteur de bloc, l'état de synchronisation et la pointe du mempool. Consultez les journaux de l'application directement dans l'interface.
- **Profils Multi-Nœuds** : Exécutez et gérez plusieurs nœuds sur la même machine. Lokinode suit automatiquement vos répertoires de données et configurations, vous permettant de basculer rapidement entre différents profils de nœuds.

## Installation et Configuration

:::tip[Vous venez de tWallet ?]
Si vous utilisez déjà [**tWallet**](/wallets/twallet), **aucune migration n'est nécessaire**. Votre phrase de récupération de 24 mots est entièrement compatible. Mieux encore, comme les deux applications partagent les mêmes répertoires de données par défaut, Lokinode détectera automatiquement votre nœud existant. Vous pouvez simplement lancer l'application et allumer votre profil de nœud par défaut **sans aucune configuration**.
:::

1. **Téléchargement** : Téléchargez l'exécutable correspondant à votre système d'exploitation.
2. **Configuration du nœud** : Lancez Lokinode. L'assistant de configuration vous demandera d'abord de définir un alias public et de sélectionner un répertoire de données local.
3. **Configuration du portefeuille** : Choisissez de créer un nouveau portefeuille (qui fournit une nouvelle phrase de récupération de 24 mots) ou de restaurer un portefeuille existant via la mnémotechnique.
4. **Sécurité** : Définissez un mot de passe de cryptage fort pour protéger vos données locales.
5. **Synchronisation** : Une fois le nœud démarré, il commencera automatiquement à se synchroniser avec la blockchain Flokicoin.

:::info[Pour les Opérateurs de Nœuds & Développeurs]
Cherchez-vous à déployer un nœud sans interface (headless) sur un VPS, à exécuter des opérations automatisées (Ops) ou à vous intégrer avec Docker ? Vous préférerez peut-être exécuter le [daemon **`flnd`**](/wallets/flnd) directement au lieu de l'interface graphique de Lokinode. `flnd` fournit le backend central et peut être entièrement géré via la ligne de commande.
:::

## Gestion avec Lokihub

Bien que Lokinode gère l'exécution de votre nœud Lightning en arrière-plan et fournisse une interface pour les opérations de base de portefeuille on-chain, vous devez utiliser **Lokihub** pour les fonctionnalités avancées du Lightning Network.

Lorsque vous ouvrez Lokihub sur la même machine, il tentera automatiquement de se connecter à votre répertoire Lokinode actif, déverrouillant les capacités suivantes :

- **Gestion de la liquidité** : Ouvrez, fermez et surveillez les canaux Lightning pour gérer votre capacité entrante et sortante.
- **Sous-portefeuilles** : Créez des portefeuilles dédiés pour organiser vos fonds pour différentes applications ou budgets personnels.
- **Intégrations d'applications (NWC)** : Connectez des applications externes à votre nœud en toute transparence à l'aide de Nostr Wallet Connect, avec des contrôles d'autorisations granulaires.

[Voir le Guide de Configuration de Lokihub →](/lokihub/setup)

## Emplacements des Données

Par défaut, Lokinode stocke les données de la blockchain et du portefeuille dans les répertoires suivants :

| OS | Chemin par défaut |
| :--- | :--- |
| **Linux** | `~/.flnd/` |
| **macOS** | `~/Library/Application Support/Flnd/` |
| **Windows** | `%APPDATA%\Flnd\` |

Vous pouvez spécifier un chemin de données personnalisé lors de la configuration initiale ou dans les paramètres de l'application.

## CRITIQUE : Sauvegardes Lightning

Votre phrase de récupération de 24 mots **ne récupère que les fonds on-chain**. 

Pour récupérer les fonds verrouillés dans les canaux Lightning ouverts, vous avez besoin d'une **Sauvegarde Statique de Canal (SCB)**. 

:::danger[RISQUE DE PERTE DE DONNÉES]
Assurez-vous toujours d'avoir une sauvegarde séparée et vérifiée de votre fichier `channel.backup` à côté de votre phrase de récupération. Sans cela, les fonds des canaux Lightning ne peuvent pas être récupérés si les données locales sont perdues.
:::
