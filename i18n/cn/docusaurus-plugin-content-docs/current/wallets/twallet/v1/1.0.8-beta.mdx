:::info[热修复更新：v1.0.8-beta.2]
**发布日期：2025 年 12 月 19 日**

已发布一个热修复版本，以解决 **Windows** 上的特定问题并更新核心依赖项。
*   **Windows 修复**：解决了键盘输入和窗口调整大小的问题（通过升级 `tcell`）。
*   **UI 稳定性**：移除了 TUI 调整大小保护，以实现更流畅的渲染。
*   **依赖项**：提升了 `flnd` 和 `tcell` 版本。

我们建议所有 Windows 用户立即升级到 **beta.2**。下方的下载链接已更新为此版本。
:::

请通过 GitHub 的问题跟踪器报告错误：  
https://github.com/flokiorg/tWallet/issues

此 Beta 版本专注于更流畅的启动/恢复流程、更清晰的入门导引、UI 精简、更强的缓存机制以及更新的依赖项。

## 变更内容

### 启动与恢复
- 添加了引导式恢复路径，当启动失败时，可清除 Neutrino 缓存文件并重启钱包服务（在启动闪屏界面按 R 键）。
- 现在启动过程会阻塞，直到服务报告健康状态；若出现宕机/EOF 情况，会显示明确的恢复或退出指令。

### 恢复与入门
- 恢复流程会监控恢复进度，显示浮窗 (toast) 更新，并在进入主应用前等待 RPC 就绪。
- 助记词创建/恢复转换过程通过跟踪恢复状态并等待钱包可用，避免了过早的页面跳转。

### 钱包 UI
- 开关/按钮增加了可配置颜色，并保持激活样式与状态同步。
- 复制密码后现在会通过浮窗确认；接收流程的提示信息更加清晰。
- 当终端报告尺寸为零时，TUI 会跳过绘制，并在返回有效尺寸后清除屏幕，防止调整大小时出现崩溃和残留帧。
- 余额页边栏始终显示未确认资金（即使存在锁定资金），以保持待处理风险的一致性。

### 配置与默认值
- 示例配置记录了 `transactiondisplaylimit`，明确其限制的是显示的交易数量（而非获取的数量）。
- CLI 默认值现在设置了交易显示限制，并明确了费率 URL 指引；示例配置记录了预期的费率 API 响应格式。

### 稳定性、缓存与监控
- 加载缓存现在会跟踪余额和链尖高度；通知会在新区块/交易产生时更新链尖。
- 恢复监控会不断重试，直到钱包 RPC 就绪并报告恢复完成。

### 依赖项
- 提升至 `flnd` **v0.1.8-beta** 和 `walletd` **v0.1.5-beta**。
- 版本号设置为 **1.0.8-beta**。

## 配置示例

```ini title="twallet.conf 代码片段"
; 限制显示的交易数量（获取的总量保持不变）
transactiondisplaylimit=50
; 推荐的费率端点格式保持不变
feeurl=https://flokichain.info/api/v1/fees/recommended
```
