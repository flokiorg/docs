Lokinode es una aplicación de escritorio multiplataforma que proporciona una interfaz gráfica para ejecutar y gestionar un nodo completo de Flokicoin (FLND). Le permite operar una cartera on-chain y participar en la Flokicoin Lightning Network (FLN).

## Características

- **Servicio en segundo plano**: Lokinode se ejecuta como un servicio persistente. Puede cerrar la interfaz de usuario mientras el nodo sigue funcionando en la bandeja del sistema.
- **Cartera On-chain**: Inicialice y desbloquee su cartera, envíe/reciba FLC y vea el historial de transacciones.
- **Gestión de nodos**: Supervise la altura de bloque, el estado de sincronización y la punta del mempool. Vea los registros de la aplicación directamente en la interfaz.
- **Perfiles Multi-Nodo**: Ejecute y gestione múltiples nodos en la misma máquina. Lokinode rastrea automáticamente sus directorios de datos y configuraciones, lo que le permite cambiar rápidamente entre diferentes perfiles de nodos.

## Instalación y Configuración

:::tip[¿Viene de tWallet?]
Si ya utiliza [**tWallet**](/wallets/twallet), **no es necesaria ninguna migración**. Su frase semilla de 24 palabras es totalmente compatible. Aún mejor, como ambas aplicaciones comparten los mismos directorios de datos predeterminados, Lokinode detectará automáticamente su nodo existente. Simplemente puede iniciar la aplicación y encender su perfil de nodo predeterminado con **cero configuración**.
:::

1. **Descarga**: Descargue el ejecutable correspondiente a su sistema operativo.
2. **Configuración del nodo**: Inicie Lokinode. El asistente de configuración primero le pedirá que establezca un alias público y seleccione un directorio de datos local.
3. **Configuración de la cartera**: Elija entre crear una nueva cartera (que proporciona una nueva frase semilla de 24 palabras) o restaurar una existente mediante mnemotecnia.
4. **Seguridad**: Establezca una contraseña de encriptación fuerte para proteger sus datos locales.
5. **Sincronización**: Una vez que el nodo se inicie, comenzará automáticamente a sincronizarse con la blockchain de Flokicoin.

:::info[Para Operadores de Nodos y Desarrolladores]
¿Está buscando desplegar un nodo sin interfaz (headless) en un VPS, ejecutar operaciones automatizadas (Ops) o integrarse con Docker? Puede que prefiera ejecutar el [daemon **`flnd`**](/wallets/flnd) directamente en lugar de la interfaz gráfica de Lokinode. `flnd` proporciona el backend central y se puede gestionar completamente a través de la línea de comandos.
:::

## Gestión con Lokihub

Mientras que Lokinode se encarga de ejecutar su nodo Lightning en segundo plano y proporciona una interfaz para operaciones básicas de cartera on-chain, debe utilizar **Lokihub** para las funciones avanzadas de Lightning Network.

Cuando abra Lokihub en la misma máquina, intentará conectarse automáticamente a su directorio Lokinode activo, desbloqueando las siguientes capacidades:

- **Gestión de Liquidez**: Abra, cierre y supervise los canales de Lightning para gestionar su capacidad de entrada y salida.
- **Sub-Carteras**: Cree carteras dedicadas para organizar sus fondos para diferentes aplicaciones o presupuestos personales.
- **Integraciones de Apps (NWC)**: Conecte aplicaciones externas a su nodo de manera transparente utilizando Nostr Wallet Connect, con controles de permisos granulares.

[Ver la Guía de Configuración de Lokihub →](/lokihub/setup)

## Ubicaciones de Datos

Por defecto, Lokinode almacena los datos de la blockchain y de la cartera en los siguientes directorios:

| SO | Ruta predeterminada |
| :--- | :--- |
| **Linux** | `~/.flnd/` |
| **macOS** | `~/Library/Application Support/Flnd/` |
| **Windows** | `%APPDATA%\Flnd\` |

Puede especificar una ruta de datos personalizada durante la configuración inicial o en la configuración de la aplicación.

## CRÍTICO: Copias de seguridad de Lightning

Su frase semilla de 24 palabras **sólo recupera los fondos on-chain**. 

Para recuperar los fondos bloqueados en canales de Lightning abiertos, necesita una **Copia de seguridad estática de canales (SCB)**. 

:::danger[RIESGO DE PÉRDIDA DE DATOS]
Asegúrese siempre de tener una copia de seguridad separada y verificada de su archivo `channel.backup` junto a su frase semilla. Sin él, los fondos de los canales de Lightning no se podrán recuperar si se pierden los datos locales.
:::
